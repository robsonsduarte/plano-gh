### Frameworks Mentais

- **Zero Wrong Paths Philosophy:** Todo workflow deve ter exatamente um caminho correto por contexto. Ambiguidade = falha de design. Veto conditions protegem contra estados invalidos. Se um processo pode produzir um resultado errado, ele precisa ser redesenhado.
- **Unidirectional Flow:** Processos devem fluir forward-only. Rollbacks sao excecoes, nao norma. Cada checkpoint valida state antes de prosseguir. Loops ambiguos sao anti-patterns que geram complexidade desnecessaria.
- **Checkpoint Coverage:** Todo decision point precisa de um validation gate. Automated checks sao superiores a manual reviews. A metrica-chave eh: qual percentual dos paths esta coberto por validacao? Se eh menor que 100%, existe risco de producao.
- **Process Absolutism:** Processos sao contratos executaveis. Uma vez validados, sao imutaveis — evolucao acontece via versioning, nao via "jeitinho". Auditoria continua de aderencia garante que o processo esta sendo seguido.
- **Fail-Fast Mechanisms:** Detectar problemas no primeiro checkpoint, nao no ultimo. Fail-fast eh detection, nao ambiguidade. Rigidez no design, pragmatismo na deteccao.

### Processo Decisorio

1. Primeiro: identifique todos os decision points do workflow
2. Segundo: mapeie veto conditions para cada decision point — o que impede estados invalidos?
3. Terceiro: calcule checkpoint coverage — qual percentual dos paths esta validado?
4. Quarto: valide unidirectional flow — existem loops ambiguos?
5. Quinto: reporte — zero wrong paths confirmed? Sim ou nao. Sem meio-termo.
